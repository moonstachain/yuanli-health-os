把身体数据，变成每日决策。

原力健康 OS 是一个 AI 智能体原生的个人健康操作系统：它不只是展示体检结果，而是把睡眠、Zone2、体成分、代谢指标和复查节奏，转化为今天的行动、每周的复盘和季度的健康实验。

进入 PC Agent 演示查看战略规划
Today

今日 Agent 简报

每天只回答三个问题：今天身体处于什么状态、今天最重要的动作是什么、今天该推进还是降载。

Protocol

Zone2 + 睡眠稳定性

把 Zone2 和睡眠锚点设为 P0 行动协议，用可执行细节推动代谢修复。

Recursive

N-of-1 实验系统

每 14 天测试一个变量，形成个人因果地图，而不是长期依赖泛泛健康建议。

战略规划

原力健康 OS 的长期方向，是从静态看板升级为一个可递归进化的个人健康 Agent：数据输入标准化，分析层形成个人基线，输出层变成每日行动和医生沟通包。

v1：外显网站

公开展示产品理念、交互风格与 Demo，不放个人隐私数据。

v2：本地数据层

Codex 解析 Apple Health、体检报告、体脂秤，生成私有 health.json。

v3：Agent 决策层

生成今日简报、Zone2/睡眠处方、缺失数据队列、医生沟通包。

v4：递归实验层

14 天实验、个人基线、相关性分析和季度健康战略会。

公开仓库原则

Public 仓库适合放产品壳、页面、schema、示例数据和文档；真实健康数据应放在私有仓库、本地文件或加密数据源中。

Privacy First

不把真实体检数据发布到公网

GitHub Pages 网站即使仓库是 private，在某些计划下页面仍可能公开访问。公开站点必须只使用 demo 数据。

Deployment

GitHub Pages 静态部署

本仓库可直接用 Pages 从 main 分支根目录发布。正式数据接入时，让前端 fetch 私有 API 或本地 data/health.json。
