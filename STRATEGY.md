# 原力健康 OS｜战略规划

## 一句话定位

原力健康 OS 不是健康数据看板，而是个人身体经营系统：每天给出行动建议，每周做经营复盘，每季生成健康实验和医生沟通包。

## 第一性原理

身体不是单一体重问题，而是能量输入、肌肉吞吐、肝脏处理、胰岛素敏感性、恢复能力与长期复查构成的动态系统。

## 产品战略

### v1 Public Site

- 建立独立 public repo
- 发布 GitHub Pages 外显网站
- 展示理念、Demo 与 Codex 接入路线
- 不公开真实健康数据

### v2 Data Layer

- 建立 `data/health.json` 标准结构
- 解析 Apple Health XML
- 解析体检 PDF / 图片报告
- 标准化体脂秤和手工记录

### v3 Agent Layer

- 今日 Agent Brief
- Zone2 处方
- 睡眠稳定性处方
- 缺失数据队列
- 医生沟通包

### v4 Recursive Health OS

- 个人基线
- 14 天 N-of-1 实验
- 行为-指标相关性分析
- 周报 / 月报 / 季度健康战略会

## P0 模块

1. Zone2：每周 3 次，每次 35–45 分钟，从体感和心率双重校准。
2. 睡眠稳定性：固定上床窗口、固定起床窗口、睡前 90 分钟降落协议。
3. 代谢修复：腰围、TG、空腹血糖、尿酸、ALT 联动管理。
4. 数据安全：public repo 只放示例，真实数据私有化。

## 成功标准

- 用户每天 1 分钟知道今天怎么做。
- 每周 10 分钟知道本周最大杠杆。
- 每 90 天知道体检指标和行为改变之间的关系。
- 所有健康建议可追溯到数据和个人基线。
